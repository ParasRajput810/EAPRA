# Contributing to EAPRA

Thank you for your interest in EAPRA. The project is in its early foundation
phase, and contributions of all kinds  issues, discussion, docs, and decisions 
are welcome.

> **Note:** The full code-contribution guide (dev setup, running tests, adding a
> plugin) will land alongside the walking skeleton. Until then, this guide covers
> how to engage with the project today.

## Ways to contribute now

- **Open an issue** to report a problem, ask a question, or propose an idea.
- **Improve the documentation** in [`docs/`](./docs).
- **Discuss architecture**  see how decisions are recorded in [`docs/adr/`](./docs/adr).

## Developer Certificate of Origin (DCO)

All commits must be signed off. This certifies you have the right to submit your
contribution under the project's license. Add the sign-off automatically with:

    git commit -s -m "your message"

This appends a `Signed-off-by: Your Name <your@email>` line to the commit. Pull
requests with unsigned commits cannot be merged.

## How decisions are made

Significant technical decisions are recorded as Architecture Decision Records
(ADRs) using the MADR template in [`docs/adr/`](./docs/adr). If you propose a
change with architectural impact, expect it to be captured as an ADR.

## Code of Conduct

All participation is governed by our
[Code of Conduct](./CODE_OF_CONDUCT.md). By contributing, you agree to uphold it.

## Pull request basics

1. Fork the repository and create a branch for your change.
2. Make your change with clear, signed-off commits (`git commit -s`).
3. Open a pull request describing what and why.
4. A maintainer will review. Discussion and iteration are normal and welcome.
